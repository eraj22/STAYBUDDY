StayBuddy Backend / AI Sticker Checklist
Scope: backend + AI only. Frontend excluded.

Legend
- [x] Implemented
- [~] Partial
- [ ] Pending

[x] Core backend modules already implemented
- [x] Node API: owner OTP flow, auth, hostel/user/review/complaint/interaction endpoints, hybrid recommendations, basic chat, warden dashboard.
- [x] AI recommendation service: FastAPI /recommend service with hybrid scoring.
- [x] Complaint AI service: FastAPI categorization, pattern detection, batch analysis.
- [x] Room matcher service: roommate and room matching endpoints.
- [x] Database schema: hostels, bookings, complaints, interactions tables are defined.

[x] Student use cases

[x] UC-1 Search Hostels Using GPS Location
- GET /api/hostels/search/nearby accepts latitude, longitude, and optional radius_km.
- Returns nearby hostels with calculated distance_km, ordered nearest first.

[x] UC-2 Search Hostels by University Name
- GET /api/hostels/search/by-university provides case-insensitive university filtering.
- Hostels support optional university, latitude, and longitude data.

[~] UC-3 Apply Basic Price Filter
- AI recommendation service accepts budget_max.
- Missing a general hostel search filter endpoint for price-only browsing.

[~] UC-4 Apply Advanced Filters (Amenities, Location, Preferences)
- AI recommendation service supports budget, distance, study, food, room type, comfort, noise, curfew, transport, and must-have amenities.
- GET /api/hostels/search/filter provides city, university, available-capacity, and text filters.
- Amenity-specific browsing filters remain pending.

[x] UC-5 AI-Powered Smart Recommendations
- Supported by /api/recommendations in Node and /recommend in the Python AI service.
- Hybrid content + collaborative scoring is already wired.

[ ] UC-6 Voice Search for Hostels
- No backend voice-to-text, intent parsing, or speech search API.

[x] UC-7 View Hostel on Map & Explore Nearby
- Nearby search responses include coordinates and calculated distance for map clients.

[x] UC-8 Save Hostels to Favorites/Wishlist
- Favorites table and add, remove, and list endpoints are available.

[x] UC-9 View Complete Hostel Details
- /api/hostels/:id returns hostel details with reviews.

[x] UC-10 Compare Multiple Hostels Side-by-Side
- GET /api/hostels/compare?ids=1,2 returns selected hostels in request order.
- Response includes location, capacity, availability, and aggregate review data.

[x] UC-11 Book Hostel & Select Room
- Rooms have hostel-scoped capacity, type, and room number data.
- Booking room assignment, availability lookup, cancellation, and reactivation keep room capacity synchronized.

[x] UC-12 File Complaint/Issue
- Complaint creation now calls the AI categorizer service at port 8001.
- AI category, priority, and suggestion are returned in the response.
- Warden endpoints added: PATCH /api/complaints/:id to update status, assign, and resolve.
- End-to-end complaint workflow from student filing to warden resolution is now functional.

[~] UC-13 Interact with AI Chatbot for Support
- /api/chat exists.
- Current backend reply is basic and not a full hostel-aware support bot.

[ ] UC-14 Send Emergency Alert
- No emergency alert endpoint, escalation workflow, or notification integration.
