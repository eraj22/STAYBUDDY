# **HMS Core - System Administrator Use Cases Documentation** 

## **Overview** 

This document details the comprehensive use cases for the System Administrator role in the HMS Core platform. The administrator manages the entire platform, monitors system health, handles user management, resolves disputes, generates analytics, and ensures smooth operation across all hostels. 

## **USE CASE A-1: Manage Hostel Registration & Verification** 

### **Element:** Detail 

**Name:** Hostel Owner Registration and Verification Process **ID:** UC-ADM-001 

**Description:** The system administrator reviews, verifies, and approves or rejects hostel registration requests from hostel owners. This includes document verification, facility inspection coordination, compliance checking, and quality assurance to ensure only legitimate, safe hostels are listed on the platform. 

**Actors:** System Administrator (Primary), Hostel Owner (Secondary), Verification Team (Secondary), Legal Team (Secondary), Backend Server (Secondary) 

### **Organization Benefits:** 

- Maintains platform quality and reputation 

- Ensures student safety through verified hostels 

- Prevents fraud and scam listings 

- Builds trust with parents and students 

- Creates competitive advantage through quality control 

- Provides legal protection through proper documentation 

- Enables data-driven hostel performance tracking 

### **Preconditions:** 

- Admin has system administrator credentials 

- Admin dashboard accessible 

- Hostel owner has submitted registration request 

- Document verification system operational 

- Verification team available for field inspections 

- Legal compliance checklist available 

### **Triggers:** 

- New hostel registration request received 

- Admin logs into dashboard and sees pending registrations 

- System sends daily digest of pending verifications 

- Urgent verification request from VIP hostel chain 

- Renewal verification due for existing hostel
