# **USE CASE P-2: Receive Child's IN/OUT Attendance Notifications** 

**Name:** Real-time Child Entry/Exit Notifications 

**Description:** The parent receives automatic push notifications, SMS, and/or email alerts whenever their child enters or exits the hostel premises. **Actors:** Parent (Primary), Student (Secondary), Biometric/RFID System (Secondary), Security Guard (Secondary), Notification Service (Primary), Backend Server (Secondary) 

# **Organization Benefits:** 

- Enhances parent trust through safety monitoring 

- Differentiates platform with premium safety features 

- Reduces parent anxiety about child's whereabouts 

- Provides accountability for hostel security systems 

- Creates detailed audit trail of student movements 

- Enables early detection of unusual patterns 

- Builds competitive advantage in parent-focused features

- Parent account must be linked to student account 

- Student must have granted attendance notification permission

- Parent must have enabled notification preferences

- Hostel security system integrated with HMS Core

- Parent opens app to view attendance history

# **Business Rules** 

# 1. **Emergency Classification:** 

- Critical (Life-threatening): Instant multi-channel alert, no delay 

- High (Serious but stable): Immediate alert within 1 minute 

- Medium (Needs attention): Alert within 5 minutes 

- Low (Informational): Standard notification 

# 2. **Notification Hierarchy:** 

- Critical emergencies: Primary parent + Secondary parent + Emergency contacts simultaneously 

- High emergencies: Primary parent first, escalate if no response in 3 minutes 

- Medium emergencies: Primary parent, escalate if no response in 10 minutes 

# 3. **Auto-Escalation Rules:** 

- No acknowledgment in 5 minutes → Call secondary contacts 

- No acknowledgment in 10 minutes → HMS Support intervenes 

- Critical emergency with no response in 15 minutes → Local authorities contacted 

# 4. **Update Frequency:** 

- Active critical emergency: Updates every 2 minutes or on significant development 

- Active non-critical: Updates every 5-10 minutes 

- Resolved emergency: Final update within 1 hour 

# 5. **Privacy & Information Sharing:** 

- Emergency details shared only with verified parents/guardians 

- Other students' parents receive only general safety info (not specific details) 

- Medical information shared only with treating medical staff and parents 

# 6. **Documentation Requirements:** 

- All emergencies must have: Timeline, Actions taken, Personnel involved, Outcome 

- Critical emergencies must include: Medical reports, Photos (if applicable), Witness statements 

- All documentation auto-archived for minimum 7 years 

# 7. **Legal & Insurance Integration:** 

- Accidents: Auto-notify insurance within 24 hours 

- Police matters: FIR numbers tracked in system 

- Medical emergencies: Hospital records linked to student profile
